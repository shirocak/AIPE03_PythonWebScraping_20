# Assignment 01: 動態網頁爬蟲實作 - マンガ図書館Z

## 📖 前言與作業背景

在進行技術開發與資料爬取時，我們必須重視一個核心議題：「技術如何應用於實際工作與生活，以及如何平衡創作者權益？」

根據統計，許多盜版漫畫與小說網站擁有極高的瀏覽量。這衍生出一個嚴肅的問題：**身為技術人員或讀者，你贊同盜版侵蝕正版方應得的利益嗎？如果不認同，你會採取什麼行動來保護正版？**

以我個人為例，我在研究所時期完成了人生第一份心血創作（論文），即便在部分教授眼中那可能只是「學術垃圾」，我依然設立了五年不公開的限制，並秉持著「這是我的垃圾別碰」的心態保護它。同樣地，目前課程的講義皆標示著：「版權所有，翻印必究。一旦發現侵權行為，將委由律師逕行提告，絕不寬貸。」

為了探討這個議題，本次作業將帶領大家認識一個誓言消滅盜版漫畫的日本網站——**「マンガ図書館Z」（漫畫圖書館Z）**。
該網站由知名漫畫家赤松健創立，其運作模式非常特別：**任何人都可以上傳擁有的盜版漫畫檔案至該平台，供使用者線上閱讀。而網站產生的廣告收益，將 100% 由原作家領取。** 這個機制不僅保存了絕版漫畫，更讓作家獲得應得的收益，甚至連發行人也親自上傳了自己作品（如：《A.Iがどまらない！》）的盜版檔案。

本次作業，我們將實作爬取該網站上的漫畫《房子機械狗》，學習如何透過動態爬蟲技術下載影像資料。

## 🏆 評分標準與要求

### 基礎目標 (80 分條件)
1. **使用 Selenium**：必須使用 Selenium 進行動態網頁資料爬取。
2. **完成程式碼填空**：修改並補齊 `Assignment01_動態爬蟲案例_mangaz_練習版` 中的程式碼。
3. **建立資料夾存檔**：程式執行時，必須能自動（或手動）新增一個資料夾，用以存放爬取下來的漫畫圖片。
4. **錄製執行結果影片**：
   * 錄製一段影片，展示您將下載下來的漫畫圖片以「投影片」或「圖片瀏覽器」方式播放的過程。
   * **不需要**展示程式碼，也**不需要**邊寫邊執行，只需展示最終下載成功的漫畫閱讀畫面。
   * 將影片上傳至 YouTube 或 Google Drive（請確認共用權限設定為「知道連結的人均可觀看」）。

### 進階目標 (100 分條件)
*必須先滿足所有 80 分條件，並追加以下要求：*

1. **程式碼模組化**：將老師提供的程式碼邏輯包裝成 `def` (函式)，並整理輸出為一個標準的 `.py` 檔案。
2. **專案結構要求**：使用 GitHub 平台來提交作業。您的 Repository 內至少需要包含：
   * `project_gutenberg` 資料夾
   * 您的 `.py` 或 `.ipynb` 執行檔
   * 本 `README.md` 說明文件
3. **提交作業**：將 **GitHub Repo 連結** 以及 **影片連結** 透過 Email 寄給老師。
